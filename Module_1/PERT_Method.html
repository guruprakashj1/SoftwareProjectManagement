PERT Method Explained

Program Evaluation and Review Technique

What is PERT?

PERT (Program Evaluation and Review Technique) is a statistical tool used in project management to analyze and represent the tasks involved in completing a project. It was developed in the 1950s by the U.S. Navy for the Polaris submarine missile program.

Key characteristics of PERT:

  • Uses probabilistic time estimates (optimistic, most likely, pessimistic)
  • Focuses on event-oriented planning
  • Identifies the critical path (longest sequence of dependent tasks)
  • Helps manage uncertainty in project scheduling

PERT Time Estimation

PERT uses three time estimates for each activity:

Optimistic Time (O): Minimum possible time required under ideal conditions
Most Likely Time (M): Best estimate of time required under normal conditions
Pessimistic Time (P): Maximum possible time required under worst-case conditions

The expected time (TE) is calculated using the formula:

TE = (O + 4M + P) / 6

The standard deviation (σ) for the estimate is calculated as:

σ = (P - O) / 6

Example Calculation

For an activity with:
Optimistic time (O) = 5 days
Most likely time (M) = 7 days
Pessimistic time (P) = 12 days

Expected time (TE) = (5 + 4×7 + 12) / 6 = (5 + 28 + 12)/6 = 45/6 = 7.5 days

Standard deviation (σ) = (12 - 5)/6 = 7/6 = 1.17 days

PERT vs. CPM Comparison

Feature PERT CPM
Time Estimation Probabilistic (3 estimates) Deterministic (1 estimate)
Focus Time (uncertainty management) Time-cost tradeoff
Best For R&D projects, uncertain durations Projects with known durations
Critical Path Identifies longest path Identifies longest path

PERT Analysis Steps

  1. Identify activities: List all tasks required to complete the project
  2. Determine sequence: Establish dependencies between activities
  3. Estimate times: Get O, M, P estimates for each activity
  4. Calculate expected times: Use PERT formula for each activity
  5. Construct network diagram: Draw activities and dependencies
  6. Identify critical path: Find the longest path through the network
  7. Update as project progresses: Revise estimates with actual data
