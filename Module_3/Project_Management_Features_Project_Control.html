Project Management Features: Project Control

Course Code: CSE4033 | Module 3.2 | Duration: 1.5 Hours

This section focuses on monitoring and controlling project execution to ensure objectives are met within constraints.

Understanding Project Control

Project control involves tracking, reviewing, and regulating progress to meet performance objectives. Key aspects include:

  • Performance Measurement: Comparing actual performance against the project plan
  • Change Control: Managing modifications to project scope, schedule, or budget
  • Quality Control: Ensuring deliverables meet requirements
  • Risk Control: Monitoring identified risks and identifying new ones
  • Reporting: Communicating project status to stakeholders

3. Change Control Process

Change Request
Impact Analysis
Approval
Implementation

Change Request: Document the requested change including:

  • Description of change
  • Reason for change
  • Requested by
  • Date requested

Impact Analysis: Assess the effect on:

  • Project scope
  • Schedule
  • Budget
  • Resources
  • Quality
  • Risks

Approval Process:

  • Review by Change Control Board
  • Decision to approve, reject, or defer
  • Documentation of decision
  • Communication to stakeholders

Implementation:

  • Update project plans
  • Assign implementation tasks
  • Monitor implementation
  • Verify completion
  • Update documentation

Scenario: Your team is developing a corporate website with the following details:

  • Budget: $100,000
  • Duration: 12 weeks

  • Planned value at this point: $50,000
